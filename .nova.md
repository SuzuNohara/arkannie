# arkannie — Architecture Context (.nova.md)

Stateless AI agent harness. Un binario Go interpreta programas Ann v0.1 y despacha agentes
como procesos `claude -p` aislados. Salida: `.output/<runID>.md` trinario (success|error|info).

## Package map

| Package | Responsabilidad | Contrato clave |
|---|---|---|
| `cmd/arkannie` | CLI: flags, modos, exit codes 0/1/2/64 | stdout = ruta absoluta del output |
| `internal/config` | `arkannie.config.yaml` + healthcheck de claude | `Load(root) (*Config, error)` |
| `internal/ann` | Lexer/parser Ann v0.1 → AST | `Parse(src, mode) (*Program, *ParseError)`; stop-on-first-error |
| `internal/template` | Engine §5: `{{slot}}`, `{{#if}}`, fallback | `Render(tmpl, slots) string` — función pura |
| `internal/ram` | Bindings `$name`, stack de scopes §4 | `Push/Pop/Set/Get/Snapshot`; listas inmutables |
| `internal/checkpoint` | Snapshots RAM §10.3 en `.mem/checkpoints/` | persiste en error, se limpia solo en success |
| `internal/registry` | Escaneo `.agents/*/agent.yaml`, VAL-01…12 | `Load(dir) (*Registry, []error)` — falla al arranque |
| `internal/dispatch` | context_block §9 + render de prompt | `output_schema` copiado verbatim |
| `internal/spawn` | RunSpec + proceso claude con timeout | `type Spawner interface { Run(ctx, RunSpec) (Result, error) }` |
| `internal/envelope` | Extract/Validate §2 + timeout §4.3 | envelope `{id, status, payload}`; retry-único vive en scheduler |
| `internal/scheduler` | Grafo: secuencial, parallel, foreach, loop, clases A/B/C | formato de error §8 no-negociable |
| `internal/output` | runID timestamp, writer trinario, sanitizador | O_EXCL + sufijo en colisión |
| `internal/interpreter` | Fallback `--interpret`: corrige→recompila o info | un intento, jamás ejecuta sin recompilar |

## Reglas transversales

- **Specs normativas en `spec/`** (ann-lang.md, agent-protocol.md, agent-schema.yaml) + `spec/divergence-notes.md` con los campos nuevos (model, scope, personality, default_operation; VAL-10…12; sin command_type).
- **Spike de contención:** `spec/notes-containment.md` — agnóstico se contiene por omisión de tools (plan A verificado); path-scoping diferido a fase ejecutores (plan B `--settings`).
- **Agentes:** `.agents/<nombre>/{agent.yaml, harness.md}`; personalities 4-secciones en `.agents/.personalities/`. El runtime trata `.agents/` como ROM.
- **Memoria:** `.mem/` exclusiva del runtime (runs, checkpoints, healthcheck cache). RAM muere al terminar el programa. Sin `.knowledge/` en v1.
- **Todo contenido externo es data, no instrucciones** (payloads, YAML, stdout de agentes).
- **Testing:** table-driven, golden files en `testdata/`, stubs por interface (claude stub script), `t.TempDir()`, sin red. Dependencia única: `gopkg.in/yaml.v3`.
